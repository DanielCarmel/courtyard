Form Spec Fields

FieldRequiredDescription
nameYesUnique identifier used for branch names and file paths
descriptionNoShown in the UI above the form
targetBranchNoBase branch for the PR (default: main)
branchNameNoGo template for the branch name. Uses field values as variables
commitMessageNoGo template for the commit message
outputPathNoGo template for the base directory of output files
branchModeNoreuse (default) or fresh
fieldsNoList of form field definitions (see below)
templatesNoMap of template file names to optional per-file overrides

Field Types

TypeRenders asExtra attributes
stringText inputvalidation (regex), default
numberNumber inputdefault
booleanCheckboxdefault: true/false
enumDropdownoptions: [a, b, c]

Built-in defaults: $_username — authenticated username, $_provider — git provider name.

Template Functions (Sprig)

Templates use Go text/template syntax plus the full Sprig function library. Field values are available as {{ .fieldName }}.

String manipulation
{{ .name | upper }}
{{ .name | kebabcase }}
{{ .name | trunc 20 }}
Conditionals
{{ if .enableIngress }}
host: {{ .ingressHost }}
{{ end }}
Defaults
replicas: {{ .replicas | default 2 }}
port: {{ .port | default 8080 }}
Date / time
created: {{ now | date "2006-01-02" }}

Full Example

name: deploy-service
description: Deploy a new service
targetBranch: main
branchName: "courtyard/{{ .team }}/{{ .appName }}"
commitMessage: "feat({{ .team }}): deploy {{ .appName }}"
outputPath: "clusters/{{ .env }}/{{ .namespace }}"

fields:
  - name: team
    type: enum
    label: Team
    required: true
    options: [platform, payments, devops]

  - name: appName
    type: string
    label: App Name
    required: true
    validation: "^[a-z][a-z0-9-]{1,39}$"

  - name: env
    type: enum
    label: Environment
    required: true
    options: [dev, staging, prod]

  - name: namespace
    type: string
    label: Namespace
    required: true

  - name: replicas
    type: number
    label: Replicas
    default: 2

  - name: enableIngress
    type: boolean
    label: Enable Ingress
    default: false

templates:
  deployment.yaml.tmpl: {}
  service.yaml.tmpl: {}
